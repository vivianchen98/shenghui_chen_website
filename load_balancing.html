Modeling load Balancing in cloud systems

Category: Research Timeline:Sep - Nov, 2019 Keywords: Formal methods, Cloud Computing

Shenghui Chen, Zhiming Fan, Haiying Shen, and Lu Feng. Performance Modeling and Verification of Load Balancing in Cloud Systems Using Formal Methods. IEEE 16th International Conference on Mobile Ad Hoc and Sensor Systems Workshops (MASSW), 2019.

Problem
In cloud computing systems, the variation of workload in different PMs will cause load imbalance and decrease the resource utilization and energy efficiency of the system. Especially the overload for the some specific PMs will affect the performance and lead to SLA violations in long term. In order to fully utilize resources, some load balancing methods have been proposed. The basic approach of load balancing is to dynamically migrate the VMs from the overloaded PM to the lightly loaded PM to avoid the overloads in the PMs in the cloud system and then guarantee the performance of PM and long-term SLA.

Contribution

  • Building a MDP model of a simplified cloud system
  • Verifying the performance characteristics of the built model by checking a list of properties using PRISM model checker
  • Interpreting the verification results to answer the questions, and give cloud service managers/providers better understandings about the system

Modeling
In this paper, we abstracted the multiple-PM system into a two-PM system, where one migration_pm migrates VMs out, and the other accepting_pm decides if it should accept the migrated VM. We also consider the workload states to be a comprehensive index of different resource types, specifically three states {High, Medium, Low}.

Verification
Multiple performance properties are checked in PRISM model checker

Takeaways

  • With enough time, the migration algorithm can guarantee the occurrence of both PM on medium-load state at the same time, but must allow both PMs to overload some time in the execution.
  • The starting status of PMs does not significantly affect the performance of the system, so the service managers do not need to worry about the timing of starting migrations.
  • One can quantify the relationship between the confidence level, requirement on performance, and the total number of PMs in the system by investigating the percent of time each PM overloads.
