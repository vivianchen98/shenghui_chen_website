Towards Personalized Explanation of Robotic Planning via User Feedback

Category: Research Timeline: Aug - Nov, 2020 Keywords: Formal methods, xAI

Kayla Boggess, Shenghui Chen, and Lu Feng. Towards Personalized Explanation of Robotic Planning via User Feedback. IEEE International Conference on Robotics and Automation (ICRA), 2021. (under review)

Problem
Prior studies have found that providing explanations about robots’ decisions and actions help to improve system transparency, increase human users’ trust of robots, and enable effective human-robot collaboration. Different users have various preferences about what should be included in explanations. How to generate personalized explanation in response to different user preferences?

Contribution

  • Developed a system for generating personalized explanations of robotic planning based on user preferences
  • Equipped the system with capabilities to interact with users for feedback, answer users’ questions about the generated explanations, and detect and resolve user preference conflicts
  • Conducted a user study with 88 users via Amazon Mechanical Turk showing encouraging results

Approach


Robotic Planning
A robot in 5 × 5 grid map;
Start at grid 20, move along arrows;
Assume move in intended direction with prob 0.8, and otherwise 0.2;
Rewards: distance (# grids), Safety (# red grids).


User Preferences
We use a tuple $\rho = (ob,lo,sp,ab)$ to represent the user preference, which includes the following four elements:

  • Objective (ob) is that of example (e.g., finding the shortest/safest route)
  • Locality (lo) describes the segment(s) of the robotic plan that the user is interested in. (e.g., the entire map/ a region of the map)
  • Specificity (sp) indicates the level of detail to include in the explanation (e.g., critical states/every state)
  • Abstraction (ab) determines the corpus of the explanation (e.g., move west in grid 12/move along the corridor in the landmark)


Generating Explanation
Input: a user preference, an optimal robotic plan, an MDP, a structured language template;
Output: an ordered list of sentences


Preference Conflict Detection and Resolution
Between the updated preference $\rho’ = (ob’, lo’, sp’, ab’)$ and original preference $\rho = (ob, lo, sp, ab)$ the system detects:

  • Soft conflict if (ob’ = ob) but (lo’ ≠ lo ∨ sp’ ≠ sp ∨ ab’ ≠ ab)
  • Hard conflict if (ob’ ≠ ob)

User study

Experiments

  • 88 participants on Amazon Mechanical Turk
  • Users allowed to input, update preferences
  • Independent: explanation type (basic vs personalized)
  • Dependent: user understanding and satisfaction

Conclusions

  • Most users update preferences, 60.23% updated at least once;
  • Personalized explanation based on user preferences increases user satisfaction, but does not improve user understanding of explanations in a statistically significant manner;
  • 82.95% liked the question-answering feature;
  • 93.18% liked the conflict detection/resolution capability.
